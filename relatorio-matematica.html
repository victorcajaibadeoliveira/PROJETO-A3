Seção 1
Introdução ao Projeto Tech Wear

O Tech Wear é uma plataforma de e-commerce especializada em roupas esportivas, desenvolvida como projeto acadêmico para a disciplina de Matemática Aplicada da Universidade São Judas Tadeu (USJT). A aplicação foi construída utilizando tecnologias web front-end (HTML, CSS e JavaScript puro) com persistência de dados via localStorage, e um módulo de backend em Python com FastAPI para processamento de imagens através dos modelos YOLO e CLIP.

O objetivo do projeto é demonstrar, de forma prática, como conceitos matemáticos fundamentais são aplicados em sistemas de software reais. Foram selecionadas três áreas de aplicação distintas, cada uma com formulação matemática própria:

  1. Cálculo de Frete — uso da tabela tarifária ANTT com fórmulas de custo logístico, impostos e tempo de entrega.
  2. Previsão de Estoque — aplicação de regressão linear simples para projetar demanda futura e identificar risco de ruptura de estoque.
  3. Similaridade de Cor por Distância Euclidiana — uso da geometria do espaço RGB e do algoritmo K-Means para recomendar produtos visualmente semelhantes a uma foto enviada pelo usuário.

As seções seguintes detalham cada uma dessas matemáticas: seu contexto de uso dentro do sistema, as fórmulas empregadas, os parâmetros utilizados e exemplos numéricos ilustrativos.

Seção 2 — Matemática Aplicada I
Cálculo de Frete — Tabela ANTT

2.1 Contexto de Uso no Sistema

O módulo de frete do Tech Wear implementa a metodologia tarifária da Agência Nacional de Transportes Terrestres (ANTT), baseada na resolução que define o piso mínimo para o transporte rodoviário de cargas. O cálculo está presente em dois pontos do sistema:

  • Checkout do cliente (wallet.html): ao informar o CEP, o sistema busca automaticamente a UF de destino via BrasilAPI, estima a distância rodoviária a partir de São Paulo e apresenta duas opções de entrega (Expressa por motoboy / Padrão por van).
  • Painel Operacional (frete.html): ferramenta administrativa completa, que permite simular o custo detalhado de qualquer pedido, com breakdown de impostos, pedágios, tempo de entrega e score de prioridade.

2.3 Fórmulas do Cálculo de Frete

Etapa 1 — Frete Base ANTT

FANTT = D × τ
Onde D = distância rodoviária em km; τ = R$ 3,8866/km

Etapa 2 — Valor do Motorista e Impostos

Vmotor = FANTT / 2
PIS = Vmotor × 0,0165
COFINS = Vmotor × 0,0760
ICMS = varia por UF de destino (0% a 17%)
Fimp = FANTT + PIS + COFINS + ICMS

Etapa 3 — Pedágio

P = ⌊D / 100⌋ × 20
Função piso: uma cobrança de R$20 para cada 100 km percorridos

Etapa 4 — Frete Final (com margem de segurança)

Ffinal = (Fimp × MS) + P + Tdesc
MS = 1,3015 (margem comercial de 30,15%)

Etapa 5 — Rateio por Volume do Pedido

Como o frete foi calculado para o caminhão consolidado (2,64 m³), o custo é rateado proporcionalmente ao volume do pedido:

Fpedido = Ffinal × (Vpedido / Vbase)
Vpedido = soma dos volumes de cada item do carrinho
Vbase = 2,64 m³ (volume de referência da van)

2.4 Fórmula de Tempo de Entrega

Ttotal = (D × Krota) / (v × fh) + n × ts + Tprep + Tmargem
D = distância | Krota = fator de tortuosidade da rota
v = velocidade média do veículo | fh = fator de eficiência horária
n = pedidos na fila | ts = tempo médio por parada

2.5 Score de Prioridade (Sp)

O painel operacional permite calcular a prioridade de despacho de um pedido com base em dois critérios ponderados:

Sp = (Urgência × 0,6) + (Densidade × 0,4)
Urgência: nota 0–10 do cliente | Densidade: razão massa/volume do pedido
Ambos normalizados para escala 0–1 antes da ponderação

Seção 3 — Matemática Aplicada II
Previsão de Estoque — Regressão Linear

3.1 Contexto de Uso no Sistema

O Dashboard de Analytics (dashboard.html) do Tech Wear implementa dois modelos preditivos baseados em regressão linear: um para prever a receita do próximo mês e outro para identificar risco de ruptura de estoque por produto.

A regressão linear é o modelo mais adequado para séries temporais curtas (6 a 12 meses de histórico), onde se deseja uma tendência de crescimento ou queda sem complexidade computacional excessiva, o que é ideal para um sistema front-end executado no navegador.

3.2 Regressão Linear Simples

Dada uma série temporal de valores mensais y0, y1, ..., yn-1 (receita ou quantidade de pedidos), a regressão linear determina a reta de melhor ajuste ŷ = m·x + b, onde:

m = (n·ΣXY − ΣX·ΣY) / (n·ΣX² − (ΣX)²)
b = (ΣY − m·ΣX) / n
n = número de meses | xi = índice do mês (0, 1, 2, ...) | yi = valor observado

3.3 Projeção do Próximo Mês

Para projetar o valor do mês seguinte (índice n), aplica-se a reta calculada no ponto além da série:

ŷn = m · n + b
Valor mínimo = 0 (garante que não seja negativo)

3.4 Coeficiente de Determinação R²

O R² (R-quadrado) mede a qualidade do ajuste da reta aos dados históricos. Varia de 0 a 1: quanto mais próximo de 1, mais confiável é a previsão.

R² = 1 − (SSres / SStot)
SSres = Σ (yi − ŷi)² ← soma dos resíduos ao quadrado
SStot = Σ (yi − ȳ)² ← variância total em torno da média
ȳ = média dos valores observados
Faixa de R²Classificação no sistemaInterpretação
0,75 — 1,00Alta confiançaTendência clara e consistente
0,40 — 0,74Média confiançaCrescimento com oscilações moderadas
0,00 — 0,39Baixa confiançaSérie instável; previsão é estimativa

3.5 Previsão de Ruptura de Estoque

Para cada produto cadastrado, o sistema calcula a velocidade de venda com base nos últimos 30 dias e projeta o estoque futuro:

vel30 = Σ unidades vendidas nos últimos 30 dias
veldia = vel30 / 30
diasrestantes = ⌊estoqueatual / veldia⌋
estoque30d = estoqueatual − vel30
reposição = max(0, ⌈vel30 × 1,2⌉ − estoqueatual)
Fator 1,2 = margem de segurança de 20% sobre a demanda prevista

4.2 Espaço de Cores RGB

Cada cor digital é representada por três componentes inteiros no intervalo [0, 255]: R (vermelho), G (verde) e B (azul). Um código hexadecimal como #4b5320 corresponde a R=75, G=83, B=32. Esses três valores definem um ponto num espaço tridimensional — o cubo de cores RGB.

4.3 Extração da Cor Dominante com K-Means

Antes de comparar cores, é necessário identificar a cor dominante da roupa na imagem. O sistema aplica o algoritmo K-Means sobre os pixels do crop central da imagem (região onde a roupa provavelmente está), com k = 4 clusters:

Inicialização: selecionar k centroids iniciais espaçados uniformemente
Atribuição: para cada pixel p, atribuir ao centroid c* mais próximo:
c* = argminc d(p, c) = argminc √[(Rp−Rc)² + (Gp−Gc)² + (Bp−Bc)²]
Atualização: recalcular cada centroid como a média dos pixels atribuídos
Repetir atribuição + atualização por no máximo 15 iterações

Após a convergência, o cluster com maior quantidade de pixels que não pertencem ao fundo da imagem representa a cor dominante da peça.

4.4 Distância Euclidiana no Espaço RGB

Para comparar a cor detectada na imagem com o hex_color de cada produto do catálogo, utiliza-se a distância euclidiana entre dois pontos no cubo RGB tridimensional:

d(A, B) = √[ (RA−RB)² + (GA−GB)² + (BA−BB)² ]
d ∈ [0 ; 441,67], onde dmax = √(255² + 255² + 255²) ≈ 441,67
d = 0 → cores idênticas | d = 441,67 → cores mais distantes possíveis (preto x branco)

4.5 Score de Similaridade Normalizado

A distância é convertida num score de similaridade entre 0 e 1, onde 1 significa cores idênticas:

similaridade(A, B) = 1 − d(A, B) / dmax
dmax = √(255² + 255² + 255²) ≈ 441,67

O sistema define um limiar mínimo de 0,60 (60% de similaridade). Produtos com score abaixo desse limiar são descartados e o usuário recebe a mensagem "não encontramos nada parecido". Os 3 produtos com maior score são exibidos como recomendações principais.

4.6 Subtração do Fundo

Para evitar que a cor do fundo da imagem (parede, chão) interfira na detecção, o sistema aplica um limiar de separação de 60 unidades de distância euclidiana:

pixel é "da roupa" se: d(pixelRGB, fundoRGB) > 60
fundoRGB é estimado pela média dos pixels das bordas da imagem (margem de 5px)

Somente os pixels classificados como "da roupa" são passados ao algoritmo K-Means, garantindo que a cor dominante detectada pertença efetivamente à peça de vestuário, não ao ambiente.

Exemplo numérico: Usuário envia foto de uma jaqueta verde-militar. K-Means detecta cor dominante: RGB = (71, 79, 28) → HEX = #474f1c. Produto "Jaqueta Windbreaker" tem hex_color = #4b5320 → RGB = (75, 83, 32). d = √[(71−75)² + (79−83)² + (28−32)²] = √[16+16+16] = √48 ≈ 6,93. similaridade = 1 − 6,93/441,67 = 98,4% → produto exibido como recomendação #1.

Conclusão
